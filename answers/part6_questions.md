# Часть 6: Контрольные вопросы

## 1) В каком порядке выполняются конструкторы при создании объекта подкласса?
_Ваш ответ_:
### Сначала вызывается конструктор суперкласса, потом — подкласса.
```class A { A() { System.out.println("A"); } }
class B extends A { B() { System.out.println("B"); } }

new B();
// A
// B
```

## 2) Можно ли переопределить `static` метод? Что такое method hiding?
_Ваш ответ:_
### Static‑методы не переопределяются, а скрываются (method hiding). Static вызывается по типу ссылки, а не объекта.
```
class A { static void f() {} }
class B extends A { static void f() {} } // скрыто
```
## 3) Что произойдет, если в `switch` нет `default` и ни один `case` не совпал?
_Ваш ответ:_
### Ничего не произойдёт — switch просто пропустится.

## 4) Почему в `for-each` нельзя удалять элементы коллекции? Как правильно?
_Ваш ответ:_
### for‑each использует итератор, и прямое удаление ломает его
### Правильно:
```
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (...) it.remove();
}
```
## 5) Чем отличается `break` с меткой от обычного `break`?
_Ваш ответ:_
### Обычный break выходит из ближайшего цикла.
### break с меткой выходит из указанного внешнего цикла.
```
outer:
for (...) {
    for (...) {
        break outer; // выходит из обоих циклов
    }
}
```
## 6) Что означает `@Override`?
_Ваш ответ:_
### Аннотация показывает, что метод переопределяет метод суперкласса или интерфейса.

## 7) Какой модификатор доступа выбрать для видимости в подклассах?
_Ваш ответ:_
### protected — виден в подклассах и в том же пакете.

## 8) Разница между `final` классом и `sealed` классом?
_Ваш ответ:_
### final — нельзя наследовать вообще.
### sealed — можно наследовать, но только разрешённым классам.

## 9) Можно ли иметь несколько конструкторов?
_Ваш ответ:_
### Да, это перегрузка конструкторов.
```
class A {
    A() {}
    A(int x) {}
}
```

## 10) Что такое ковариантный тип возвращаемого значения?
_Ваш ответ:_
### Переопределённый метод может возвращать более конкретный тип, чем в суперклассе.
```
class A { A clone() { ... } }
class B extends A { @Override B clone() { ... } }
```
